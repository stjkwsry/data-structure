最小生成树

生成树的基本概念

生成树指的是所有顶点均由边连接在一起，但不存在回路的图。

一个图可以有许多棵不同的生成树，所有的生成树具有以下共同特点：

  • 生成树的顶点个数与图的顶点个数相同。
  • 生成树是图的极小连通子图，去掉一条边则非连通。
  • 一个有 nnn 个顶点的连通图的生成树有 n−1n-1n−1 条边。
  • 在生成树中再加一条边必然会形成一条回路。
  • 生成树中任意两个顶点间的路径是唯一的。

含有 nnn 个顶点 n−1n-1n−1 条边的图不一定是生成树。

如果给定一个无向网，在该网的所有生成树中，使得各边权值之和最小的那棵生成树就称为该网的最小生成树，也叫最小代价生成树。

例如，需要在 nnn 个城市之间建立通信网，则 nnn 个城市应该铺设 n−1n-1n−1 条线路，但因为每条线路都会有对应的经济成本，而 nnn 个城市最多有 n(n−1)/2n(n-1)/2n(n−1)/2 条路线，那么如何选择 n−1n-1n−1 条路线，才能使总费用最少？

将这个现实问题抽象出数学模型可以得到：

现实事物 图结构
nnn 个城市 nnn 个顶点
n−1n-1n−1 条线路 n−1n-1n−1 条边
线路的经济代价 边的权值
城市间的通信网 连通网

很显然这个连通网就是一个生成树，解决此问题的关键就是如何求得最小生成树。

在生成树的构造过程中，图中 nnn 个顶点分属于两个集合：

  • 集合 UUU - 已经落在生成树上的顶点集。
  • 集合 V−UV-UV−U - 尚未落在生成树上的顶点集。

接下来则应在所有连接 UUU 中顶点和 V−UV-UV−U 中顶点的边中选取权值最小的边加入到生成树中，且加入这条边之后生成树不会存在回路。

Prim 算法

普里姆算法（Prim 算法）是图论中的一种算法，可以在加权连通图中搜索最小生成树，它的算法思想如下：

  1. 设 N=(V,E)N=(V,E)N=(V,E) 是连通网，TETETE 是 NNN​ 上最小生成树中的边的集合。
  2. 初始令 U={u0}(u0∈V)U=\{u_0\}(u_0{\in}V)U={u​0​​}(u​0​​∈V)，TE={∅}TE=\{\emptyset\}TE={∅}。
  3. 在所有 u∈Uu{\in}Uu∈U，v∈V−Uv{\in}V-Uv∈V−U 的边 (u,v)∈E(u,v){\in}E(u,v)∈E 中，找一条代价最小的边 (u0,v0)(u_0,v_0)(u​0​​,v​0​​)，将其并入集合 TETETE，同时 v0v_0v​0​​ 并入集合 UUU。
  4. 重复 3 的操作，直至 U=VU=VU=V 为止，则 T=(V,TE)T=(V,TE)T=(V,TE) 为 NNN 的最小生成树。

  4. 现在剩下 v2v_2v​2​​ 和 v5v_5v​5​​，显而易见的，V−UV-UV−U 与 UUU 的最小权值边为 v2−v3v_2-v_3v​2​​−v​3​​，故将 v3v_3v​3​​ 与 v2v_2v​2​​ 相连，并将 v2v_2v​2​​ 也加入到集合 UUU 中。

Kruskal 算法

克鲁斯卡尔算法（Kruskal 算法），是更直截了当的贪心算法，它的算法思想如下：

  1. 设连通网 N=(V,E)N=(V,E)N=(V,E)，令最小生成树初始状态为只有 nnn 个顶点而无边的非连通图 T=(V,{∅})T=(V,\{\emptyset\})T=(V,{∅})，每个顶点自成一个连通分量。
  2. 在 EEE 中选取代价最小的边，若该边依附的顶点落在 TTT 中不同的连通分量上（即不能形成环），则将此边加入到 TTT 中；否则舍去此边，选取下一条代价最小的边。
  3. 以此类推，直到 TTT 中所有顶点都在同一连通分量上为止。

  2. 选择未连接的最小的权值边 v4−v6v_4-v_6v​4​​−v​6​​，将这两个顶点连接，加入生成树。
  3. 选择未连接的最小的权值边 v2−v5v_2-v_5v​2​​−v​5​​，将这两个顶点连接，加入生成树。
  4. 选择未连接的最小的权值边 v3−v6v_3-v_6v​3​​−v​6​​，将这两个顶点连接，加入生成树。
  5. 现在有 3 条权值为 5 的边，分别是 v1−v4v_1-v_4v​1​​−v​4​​、v3−v4v_3-v_4v​3​​−v​4​​、v2−v3v_2-v_3v​2​​−v​3​​，依次将其连接判断：
    1. 当连接 v1v_1v​1​​ 和 v4v_4v​4​​ 时，v1−v4−v3v_1-v_4-v_3v​1​​−v​4​​−v​3​​ 形成了环路，所以不考虑连接。
    2. 当连接 v3v_3v​3​​ 和 v4v_4v​4​​ 时，v3−v4−v6v_3-v_4-v_6v​3​​−v​4​​−v​6​​ 形成了环路，所以也不考虑连接。
    3. 当连接 v2v_2v​2​​ 和 v3v_3v​3​​ 时，没有形成环路，所以将这两个顶点连接，加入生成树。
  6. 至此，生成树中已经拥有 n−1n-1n−1 条边，通过 Kruskal 算法构造出了最小生成树。
