图的遍历算法

图的遍历算法

与树的遍历算法相似，图的遍历是指从图的某个顶点出发，对图的每一个顶点访问一次，且仅访问一次的操作。

在应用中，很多图的操作都是基于图的遍历操作基础之上进行的。图的遍历操作相对比较复杂，主要体现在以下几个方面：

  1. 若要在图中设立首结点，那么图中任何一个结点都可以作为首结点。
  2. 在非连通图中，从一个顶点出发，只能够访问其所在的连通分量上的所有顶点，所以还必须考虑如何选择下一个出发点以访问图中其他的连通分量。
  3. 在图的结构中有回路存在，当一个顶点被访问之后，可能会沿回路又回到该起点。
  4. 在图的结构中，一个顶点可以和其他多个顶点相连，当这样的顶点访问过后，存在如何选择下一个要访问的顶点的问题。

总结：导致图的遍历序列不唯一的因素是：出发点不同，存储结构不同，遍历方法不同。

通常，图的遍历有深度优先搜索和广度优先搜索两种方式，下面分别做介绍。

深度优先搜索

DFS 的思想

图的深度优先搜索（Depth First Search，DFS）算法的的遍历类似于树的先根遍历，是树的先根遍历的一种推广，俗称为 DFS 算法。

假设在初始状态下，图的任何顶点都没有被访问过。深度优先搜索可以从图的某个顶点 vvv 出发，访问此顶点，然后依次从 vvv 未被访问的邻接点出发深度优先遍历图，直到图中所有和 vvv 有路径相同的顶点都被访问到为止。若此时图中还有未被访问的顶点，则另外选择图中一个未被访问的顶点作为起点，重复上述过程，直到图中所有的顶点都被访问到为止。

DFS 的示例

有图 G3G_3G​3​​：

以图 G3G_3G​3​​ 为例。假设选中 aaa 作为起始顶点，然后依次采用深度优先搜索访问 aaa 的其他邻接点。如果第二步选择访问 ggg，然后再深度访问 ggg 的其他邻接点，访问顺序就会变成：

a→g→h→b→d a\to{g}\to{h}\to{b}\to{d} a→g→h→b→d

然后再去访问 aaa 未被访问的邻接点 ccc：

a→g→h→b→d→c a\to{g}\to{h}\to{b}\to{d}\to{c} a→g→h→b→d→c

当把 aaa 的邻接点访问完了以后，G3G_3G​3​​ 还有两个连通分量没有被访问到，则选择其中一个连通分量的一个顶点 fff，然后顺序访问下去：

a→g→h→b→d→c→f→e a\to{g}\to{h}\to{b}\to{d}\to{c}\to{f}\to{e} a→g→h→b→d→c→f→e

最后再选择剩下的连通分量，以（任意）顶点 kkk 开始顺序访问，就得到了整个图的深度优先搜索访问过程：

a→g→h→b→d→c→f→e→k→j→i a\to{g}\to{h}\to{b}\to{d}\to{c}\to{f}\to{e}\to{k}\to{j}\to{i} a→g→h→b→d→c→f→e→k→j→i

DFS 的算法实现

在访问过程中，为了表示某个顶点是否被访问过，需要给每个顶点设置一个访问标记域 visit，当 visit 为 TRUE 时，表示该顶点已经被访问过，否则为 FALSE 时就表示没有被访问过。

采用递归算法很容易实现，DFS 的一般算法实现如下：

图的数据结构基于邻接表实现。

BOOL visit[MAX_VERTEX_NUM];

// 辅助函数，输出访问结点的信息，在实际应用过程中可以设置其他操作
void VisiteNode(GNode *gn) {
    printf("%s, %s\n", gn->nodeID, gn->nodeName);
}

// 返回指定结点下一个没有访问过的邻接点
int NextAdjVex(AdjacencyListGraph *alg, int i) {
    ArcNode *an = alg->vexs[i].firstArc;
    // 寻找没有访问过的邻接点
    while (visit[an->adjVex] == TRUE && an != NULL) {
        an = an->nextArc;
    }
    // 如果找到了，就返回它
    if (an != NULL) {
        return an->adjVex;
    }
    return ERROR;
}

// 深度优先从某一个顶点出发的算法
void SingleDFS(AdjacencyListGraph *alg, int i, void (*vf)(GNode *gn)) {
    // 访问标志
    visit[i] = TRUE;
    // 访问结点的函数
    vf(&(alg->vexs[i].node));
    // 递归访问邻接点
    for (int j = NextAdjVex(alg, i); j != ERROR; j = NextAdjVex(alg, i)) {
        if (!visit[j]) {
            SingleDFS(alg, j, vf);
        }
    }
}

// 深度优先搜索算法
void DepthFirstSearch(AdjacencyListGraph *alg, void (*vf)(GNode *gn)) {
    // 标志所有顶点为没有被访问
    for (int i = 0; i < alg->vexNum; i++) {
        visit[i] = FALSE;
    }
    // 开始深度优先搜索
    for (int i = 0; i < alg->vexNum; i++) {
        if (!visit[i]) {
            SingleDFS(alg, i, VisiteNode);
        }
    }
}

从算法中可以看出，使用深度优先搜索算法是对图中的每一个顶点最多调用一次 SingleDFS() 函数，因为一旦某个顶点被标记为已访问，就不再从它出发进行搜索。因此，遍历图的过程其实就是对每一个顶点查找其邻接点的过程，其时间复杂度完全取决于图所采用的存储结构。

在 DFS 算法中，既然是为了取得邻接点，所以以上算法是基于图的邻接表的存储方式实现的，当采用邻接矩阵时，图的遍历时间相较于邻接表会增多。

广度优先搜索

BFS 的思想

广度优先搜索（Breadth First Search）遍历类似于树的层次遍历的过程。

假设从图中某个顶点 vvv 出发，在访问了 vvv 之后依次访问 vvv 的各个未曾访问过的邻接点，然后分别从这些邻接点出发依次访问它们的邻接点，并使先被访问的顶点的邻接点先于后被访问的顶点的邻接点访问，直到图中所有已被访问的顶点的邻接点都被访问到。若此时图中还有顶点未被访问，则另外选择图中一个未曾被访问的顶点作为起点，重复上述过程，直到图中所有顶点都访问到为止。

BFS 的示例

有图 G3G_3G​3​​：

以图 G3G_3G​3​​ 为例。假设选中 aaa 作为起始顶点，然后依次采用广度优先搜索访问 aaa 的其他邻接点。如果第二步选择访问顶点 ggg、ddd、ccc、bbb，则有：

a→g→d→c→b a\to{g}\to{d}\to{c}\to{b} a→g→d→c→b

然后再顺序访问 ggg、ddd、ccc、bbb 的邻接点。ggg 有邻接点 ddd 和 hhh，但 ddd 已经被访问，所以只能访问顶点 hhh：

a→g→d→c→b→h a\to{g}\to{d}\to{c}\to{b}\to{h} a→g→d→c→b→h

同样，ddd 的邻接点已经全部被访问过，看向下一个顶点；ccc 的邻接点 aaa 已经被访问过，看向下一个顶点；bbb 的所有邻接点也都被访问过，这时再（任意）选择一个没有被访问过的顶点，这里选择顶点 fff，之后同样采用上面的广度优先搜索进行访问，有：

a→g→d→c→b→h→f→e a\to{g}\to{d}\to{c}\to{b}\to{h}\to{f}\to{e} a→g→d→c→b→h→f→e

最后再（任意）选择顶点 iii，（随意）以 iii、kkk、jjj 的顺序访问。这样，就得到了整个图的广度优先访问结果：

a→g→d→c→b→h→f→e→i→k→j a\to{g}\to{d}\to{c}\to{b}\to{h}\to{f}\to{e}\to{i}\to{k}\to{j} a→g→d→c→b→h→f→e→i→k→j

BFS 的算法实现

跟 DFS 算法一样，在 BFS 访问过程中，为了表示某个顶点是否被访问过，需要给每个顶点设置一个访问标记域 visit，当 visit 为 TRUE 时，表示该顶点已经被访问过，否则为 FALSE 时就表示没有被访问过。

图的数据结构基于邻接表实现。

// 使用队列存储访问过的结点
typedef struct {
    GNode Queue[MAX_VERTEX_NUM];
    int Header, Tail;
    int Count;
} GQueue;

void InitQueue(GQueue *queue) {
    queue->Count = queue->Header = queue->Tail = 0;
}

BOOL EmptyQueue(GQueue *queue) {
    if (queue->Count == 0) {
        return TRUE;
    }
    return FALSE;
}

// 入队
BOOL EnQueue(GQueue *queue, GNode *gn) {
    if (queue->Count >= MAX_VERTEX_NUM) {
        return ERROR;
    }

    CopyNode(&(queue->Queue[queue->Count]), gn);
    queue->Count++;
    queue->Tail = queue->Count;
    return TRUE;
}

// 出队
BOOL DeQueue(GQueue *queue, GNode *gn) {
    if (EmptyQueue(queue) == TRUE) {
        return ERROR;
    }

    CopyNode(gn, &(queue->Queue[queue->Header]));
    queue->Header++;
    queue->Count--;
    return TRUE;
}

BOOL visit[MAX_VERTEX_NUM];

// 广度优先搜索
void BreadthFirstSearch(AdjacencyListGraph *alg, void (*vf)(GNode *gn)) {
    // 初始化访问标志
    for (int i = 0; i < alg->vexNum; i++) {
        visit[i] = FALSE;
    }

    GQueue *queue;
    InitQueue(queue);

    // 遍历顶点
    for (int i = 0; i < alg->vexNum; i++) {
        if (visit[i] == FALSE) {
            visit[i] = TRUE;
            vf(&(alg->vexs[i].node));
            // 访问过的顶点入队
            EnQueue(queue, &(alg->vexs[i].node));

            // 查找访问过的顶点的其他邻接点
            while (EmptyQueue(queue) != TRUE) {
                GNode *gn;
                DeQueue(queue, gn);
                // 获取出队的顶点的邻接点，此函数详见 DFS 的算法实现
                int j = NextAdjVex(alg, gn->nodeID);
                while (j != ERROR) {
                    visit[j] = TRUE;
                    vf(&(alg->vexs[i].node));
                    // 访问过的顶点入队
                    EnQueue(queue, &(alg->vexs[i].node));
                }
            }
        }
    }
}

图的邻接表的广度优先遍历的时间复杂度为 O(n)O(n)O(n)，如果采用邻接矩阵的存储方式，复杂度会是 O(n2)O(n^2)O(n​2​​)。在实际应用中，应根据实际情况选择适当的存储结构，这样会使程序的算法执行效率提高很多。
