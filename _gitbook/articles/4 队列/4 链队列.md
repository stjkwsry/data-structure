# 链队列

当队列中的元素个数无法预先估计时，顺序存储结构的缺点就暴露出来了，因为 C 语言并不能用动态分配的一维数组来方便地实现顺序存储结构的队列。

在实际应用中，若无法预测所用队列的最大长度，则最好采用链式队列。

用链表来表示的队列简称为**链队列**，是限制在表头删除和表尾插入的**单链表**，一般数据结构定义如下：

```c
typedef struct QNode {
    ElemType Elem;
    struct QNode *Next;
} QNode;

// 头结点
typedef struct {
    // 队头和队尾结点指针，不使用 Elem 变量，只使用 Next 指针指向头尾结点
    QNode *Front, *Rear;
    // 记录队列中元素的个数
    int Count;
} LQueue;
```

链队列的一般算法实现如下：

```c
Status InitLQueue(LQueue &q) {
    // 为头结点分配空间
    q.Front = (QNode*) malloc(sizeof(QNode));
    // 未分配成功
    if (!q.Front) {
        printf("Memory Error\n");
        return ERROR;
    }
    // 设置首元结点为空
    q.Front->Next = NULL;
    // 头尾相等
	q.Rear = q.Front;
    q.Count = 0;
    return OK;
}

Status IsEmpty(LQueue q) {
    if (q.Front == q.Rear) {
        return TRUE;
    }
    return FALSE;
}

Status InsertLQueue(LQueue &q, ElemType e) {
    // 定义一个结点并为其分配空间
    QNode* p = (QNode*) malloc(sizeof(QNode));
    if (!p) {
        printf("Memory Error\n");
        return ERROR;
    }
    p->Elem = e;
    p->Next = NULL;
    // 设置为原队尾的后继
    q.Rear->Next = p;
    // 将新元素设置为新队尾
    q.Rear = p;
    // 记录元素个数
    q.Count++;
    return OK;
}

Status DeleteLQueue(LQueue &q, ElemType &e) {
    if (IsEmpty(q) == TRUE) {
        return ERROR;
    }
    // 定义一个结点指针 p，指向队头元素
    QNode *p = q.Front->Next;
    // 返回被删除的队头元素
    e = p->Elem;
    // 修改队头指针
    q.Front->Next = p->Next;
    // 释放原队头元素的内存
    free(p);
    // 记录元素个数
    q.Count--;
    return OK;
}

Status DestroyLQueue(LQueue &q) {
    QNode* p;
    while (q.Front) {
        p = q.Front->Next;
        free(q.Front);
        q.Front = p;
    }
}

void ListLQueue(LQueue q, void (*f)(ElemType e)) {
	QNode *p = q.Front->Next;
	while (p) {
		(*f) (p->Elem);
		p = p->Next;
	}
}
```
